# 50. セッション管理（中断・再開）

## 中断の判断基準

以下の状況で作業を中断する:
- 残り turns が少なくなったとき（目安: 残り10turns以下）
- 大きなタスクの区切りが来たとき
- エラーが続いて解決の目処が立たないとき

## 中断時にやること

1. **Issue/PR コメントに作業ログを残す**
   - 完了した作業 / 未完了の作業 / 現在の状態 / 次にやるべきこと / 仮定・判断
2. **WIP コミットをプッシュする**
   - メッセージ形式: `WIP: [作業内容の要約]`

## ブランチ命名規則

- 形式: `claude/<issue-type>-<issue-number>-<YYYYMMDD>-<HHMM>`
- 中断時の追加ブランチ: `claude/wip/<issue-number>-<連番>`

## 再開時にやること

1. Issue/PR コメントで前回の作業ログを確認（「次にやるべきこと」を最優先）
2. ベースブランチの更新を確認し、必要ならリベース
3. 前回のログを参照し、同じ調査を繰り返さない
